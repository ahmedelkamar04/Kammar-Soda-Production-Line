Question Bank

Technical questions and answers covering Git, MES, SCADA, Docker, InfluxDB, Grafana, and the KAMAR SODA project.

Question Bank

Q1. In the kitchen analogy, what does the staging area represent?

The staging area is like a preparation table. The command git add <file> moves a file into the staging area.

Q2. What does git init do?

The command git init converts a normal folder into a Git repository and creates a hidden .git directory that stores all version control information.

Q3. Explain the three-zone mental model of Git.

The Working Directory contains files being edited. The Staging Area contains files prepared for commit using git add. The Commit History stores permanent snapshots created using git commit.

Q4. What is the difference between git diff and git status?

git status shows which files have changed, while git diff displays the exact line-by-line modifications made to the files.

Q5. What does git restore <file> do?

git restore removes uncommitted changes and restores the file to its last committed state.

Q6. What is a Git branch?

A branch is an independent line of development that allows new features and experiments to be created without affecting the main branch.

Q7. What does git clone <url> do?

git clone downloads an existing repository along with its complete history. Unlike git init, it starts from an existing project rather than creating a new repository.

Q8. What is a remote in Git?

A remote is a shared repository hosted on a server such as GitHub. git push uploads local commits, while git pull downloads and integrates changes from the remote repository.

Q9. Which are the SEMI E10 Equipment States?

The SEMI E10 states are Standby, Productive, Engineering, Scheduled Downtime, Unscheduled Downtime, and Non-Scheduled Downtime.

Q10. How are the SEMI E10 time buckets represented?

Total Time is divided into Operations Time and Non-Scheduled Time. Operations Time is divided into Uptime and Downtime. Uptime includes Productive, Standby, and Engineering. Downtime includes Scheduled and Unscheduled Downtime.

Q11. What is an HMI and what are the three rules of good HMI design?

An HMI is a Human Machine Interface that allows operators to monitor and control equipment. Good HMI design should show only necessary information, make abnormal conditions obvious, and always represent the true machine state.

Q12. What is the difference between Back End and Front End?

The Back End contains the logic, data, and decision-making processes. The Front End displays information and receives user input. In this project, production_line.py is the Back End and the HMI acts as the Front End.

Q13. What is the role of the Abstract Base Class Station?

The Station abstract class defines a common interface for all production stations. Concrete subclasses such as BottleBlow, SodaFill, Carbonation, CapOn, Label, and QualityControl implement their own processing behavior.

Q14. What is the SEMI E10 standard and what states are used in the project?

SEMI E10 is a standard used to classify machine time. The KAMAR SODA uses Productive, Standby, and Unscheduled Downtime states.

Q15. What does OEE stand for?

OEE stands for Overall Equipment Effectiveness. It is calculated as Availability × Performance × Quality. A world-class manufacturing line targets an OEE above 85%.

Q16. What is a Manufacturing Execution System (MES)?

A Manufacturing Execution System monitors and coordinates factory-floor operations in real time. MES operates at ISA-95 Level 3 between production equipment and enterprise planning systems.

Q17. What is a Finite State Machine (FSM)?

A Finite State Machine allows an object to exist in only one state at a time. In this project, bottles move through states such as blowing, filling, carbonation, capping, labeling, quality control, packed, and rejected.

Q18. In the InfluxDB/Grafana stack, which component writes, stores, and visualizes data?

The Python backend writes production data, InfluxDB stores the data, and Grafana visualizes the information through dashboards and charts.

Q19. What is Docker and what are its main applications?

Docker is a containerization platform that packages applications and dependencies into portable containers. It is commonly used for deployment, testing, microservices, and cloud-native applications.

Q20. Describe the architecture of the KAMAR SODA project.

The KAMAR SODA consists of six production stations: Bottle Blow, Soda Fill, Carbonation, Cap On, Label, and QC & Pack. The Python backend controls bottle movement, state transitions, production statistics, and defect detection. A SCADA-style HMI provides operator control and monitoring. Production data is stored in InfluxDB and visualized through Grafana dashboards. Docker containers host the services, creating a complete industrial automation simulation similar to a small Manufacturing Execution System operating at ISA-95 Level 3.
