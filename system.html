The big picture

Six stations, four software layers, one intelligent production system.

The KAMAR SODA is a simulated industrial bottling line designed to demonstrate modern manufacturing principles, production monitoring and OEE calculation.

The production line consists of six stations: Bottle Blow, Soda Fill, Carbonation, Cap On, Label and QC & Pack.

Production Flow

Bottle Blow → Soda Fill → Carbonation → Cap On → Label → QC & Pack

Each station reports production counts, machine status, quality information and downtime events.

System Overview

The KAMAR SODA is organized into four layers. The physical layer models the six manufacturing stations. The Python backend controls bottle movement, state transitions, production counters, and defect detection.

The HMI provides operator controls and live process monitoring. Production data is stored in InfluxDB and visualized using Grafana dashboards.

Operator HMI

The Human Machine Interface (HMI) is the primary operator control panel for the KAMAR SODA. It provides real-time visibility of production performance, bottle counts, machine status and alarm conditions.

Operators can start, stop, reset and safely shut down the production line using dedicated control buttons. The HMI also displays the active production stage and overall system efficiency, allowing rapid response to faults and process interruptions.

Front End — HMI Design

The HMI is developed using Python and pygame. It provides real-time visibility of production counts, defect rate, alarms, machine state and production progress.

Operators can Start, Stop, Reset and Emergency Stop the production line. Defective products are identified and alarm messages indicate the cause of the fault.
