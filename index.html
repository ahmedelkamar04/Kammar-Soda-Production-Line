Soda Bottle Production Line

A simulated carbonated soft-drink bottling system with Python backend logic, a SCADA-style HMI, InfluxDB data storage, Grafana dashboards, Docker deployment, and GitHub Pages documentation.

Project overview

The project models a 500 ml soda bottle production line. The process begins with bottle forming and continues through filling, carbonation, capping, labeling, quality control, and packing.

Each station represents a real manufacturing step. Defective bottles can be detected because of underfilling, missing caps, missing labels, or low carbonation.

Operator HMI

The SparkFlow Human Machine Interface (HMI) provides operators with real-time visibility into production performance. Operators can start, stop and reset the production line while monitoring bottle counts, alarms, machine status and efficiency metrics.

Grafana Monitoring Dashboard

Production data is streamed into InfluxDB and visualized through Grafana dashboards. The dashboard displays throughput, defect rates, station status and overall production efficiency in real time.

The Python backend controls the production line logic. The SCADA-style HMI gives operators live control and monitoring. InfluxDB stores production data, and Grafana visualizes the data through dashboards.
